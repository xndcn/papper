## 1. 美术风格定义

### 1.1 整体风格

本作采用 **2D像素风（Pixel Art）** 作为核心美术风格，追求清晰、精致、富有表现力的像素画面。

**设计关键词：** 童趣、温暖、纸质感、明亮、怀旧

视觉上对标以下参考作品：

- **元气骑士（Soul Knight）** —— 流畅的像素动画表现与鲜明的角色辨识度
- **死亡细胞（Dead Cells）** —— 精致的像素精灵与动态光影效果
- **Celeste** —— 干净利落的像素场景构图与色彩运用
- **Stardew Valley** —— 温暖柔和的色调与生活气息
- **Slay the Spire** —— UI布局与地图节点设计的信息层次感

整体色调以**明亮温暖**为主基调，突出童年回忆中阳光下折纸飞机的质感。纸张的白色、米色作为核心视觉锚点，搭配蓝天、绿地、暖阳等自然色系，营造积极向上的冒险氛围。

### 1.2 像素分辨率规范

| 资源类型 | 尺寸规范 | 说明 |
|---------|---------|------|
| Tile 基础尺寸 | **32x32 px** | 推荐标准，平衡精致度与制作效率 |
| 角色精灵 | **32x32** 或 **32x48 px** | 可适当增加高度以表现角色的立体感 |
| 纸飞机精灵 | **48x24** 或 **64x32 px** | 横向长条形，符合飞机外观比例 |
| 游戏内部分辨率 | **480x270 px**（16:9） | Perfect x2 = 960x540, Perfect x4 = 1920x1080 |
| 缩放模式 | **整数倍缩放（Integer Scaling）** | 必须保持像素边缘清晰，禁止非整数缩放导致模糊 |

**关于内部分辨率的说明：** 480x270 是一个理想的基础分辨率。它是 16:9 比例，且可以通过整数倍（x2、x4）完美映射到 960x540 和 1920x1080 等主流显示分辨率，确保每个像素在屏幕上呈现为完美的正方形方块，不会出现次像素偏移的视觉瑕疵。

### 1.3 调色板规范

为保持全游戏的视觉风格统一，采用**限定调色板（Restricted Palette）** 策略。

**主调色板**

- 基础色数：**32色 或 48色**
- 推荐参考调色板：**Endesga 32** 或 **PICO-8 调色板**
- 所有场景和角色精灵原则上只从主调色板中取色，特殊情况（如UI高亮、特效粒子）可酌情扩展

**纸飞机专用色**

纸飞机的主体始终保持"纸"的质感色系，通过强调色区分不同机型：

| 用途 | 色系 |
|------|------|
| 纸张本体 | 白色、浅灰、米色、象牙色 |
| 速度型强调色 | 红色、橙色 |
| 特技型强调色 | 紫色、粉色 |
| 稳定型强调色 | 蓝色、绿色 |
| 均衡型强调色 | 黄色、绿色 |
| 传说级/Boss级 | 金色、银色 |

**UI专用色**

UI元素的配色需与游戏场景保持视觉区分，避免信息混淆：

- 面板背景：半透明深色（黑色 60%-80% 透明度）
- 文字颜色：白色（主文本）、黄色（高亮/数值）、灰色（次要信息）
- 按钮边框：白色或浅蓝，悬停态使用亮色

**天气调色滤镜**

每种天气条件叠加对应的色调滤镜，在不修改底层资源的前提下改变画面氛围：

| 天气 | 滤镜效果 |
|------|---------|
| 晴天 | 无滤镜，原始明亮色调 |
| 阴天 | 轻微去饱和 + 偏灰蓝调 |
| 雨天 | 降低亮度 + 蓝灰色调 + 雨滴粒子叠加 |
| 暴风雨 | 明显压暗 + 深蓝灰调 + 闪电白闪 |
| 黄昏 | 暖橙色调覆盖，长阴影 |
| 雪天 | 偏白蓝冷色调 + 雪花粒子叠加 |
| 大风 | 轻微偏黄 + 风线粒子特效 |

### 1.4 参考风格板（Mood Board）要点

制作参考风格板时应收集以下方向的素材：

- 阳光下儿童折纸的摄影/插画（捕捉纸的质感与光影）
- 小学校园、操场、后山、小卖部等童年场景
- 像素风赛车/飞行游戏的UI截图
- 纸飞机比赛的真实照片（了解不同机型的外观差异）
- 上述参考游戏的画面截图

## 2. 场景美术

### 2.1 节点地图（大地图）

节点地图是玩家在比赛之外进行探索、交互的主要界面，采用**鸟瞰式城镇地图**作为底图。

**设计要点：**

- 整体呈现一个温馨的小城镇/社区俯瞰图，作为底层背景绘制
- 地图上的可交互地点以**建筑图标**表示，尺寸为 **32x32 或 64x64 px**
- 地点之间用**小路**连接：
  - 实线小路：已解锁/已访问的路线
  - 虚线小路：未解锁的路线（灰色半透明）
  - 当前可前往的路线：轻微发光或闪烁提示

**地点图标设计清单：**

| 地点 | 图标元素 | 功能说明 |
|------|---------|---------|
| 家 | 温馨小房子、屋顶冒烟 | 存档、休息、查看收藏 |
| 学校 | 校舍建筑、国旗 | 教学关卡、折纸学习 |
| 后山 | 树木、山丘、小路 | 练习飞行、隐藏道具 |
| 小卖部 | 小店铺、招牌、遮阳棚 | 购买零件、材料 |
| 公园 | 秋千架、长椅、喷泉 | NPC交互、支线任务 |
| 比赛场地 | 旗帜、跑道标线 | 正式比赛入口 |
| 锦标赛会场 | 大型建筑、奖杯图标 | 锦标赛/Boss战入口 |

**昼夜变化效果：**

地图支持纯视觉性的昼夜变化，不影响玩法机制：

- 白天：正常明亮色调，蓝天白云背景
- 傍晚：暖橙色滤镜叠加，建筑投射长影
- 夜晚：深蓝色调，窗户透出暖黄灯光，星空背景

昼夜切换与游戏进程（而非真实时间）挂钩，每完成一定数量的比赛或事件后推进时段。

### 2.2 比赛背景

比赛场景为**横版滚动（Side-scrolling）** 视角，纸飞机从左向右飞行。

**背景主题列表：**

| 主题 | 视觉要素 | 色调 |
|------|---------|------|
| 蓝天白云（默认） | 晴朗天空、蓬松白云、远处山峦 | 天蓝 + 白色 |
| 城市天际线 | 建筑轮廓、水塔、天线、广告牌 | 灰蓝 + 暖光 |
| 山地丘陵 | 起伏山脉、针叶林、溪流 | 深绿 + 蓝天 |
| 海岸线 | 海面波光、沙滩、灯塔、海鸥 | 蓝绿 + 金沙 |
| 日落/黄昏 | 渐变天空（橙→紫）、剪影建筑 | 暖橙 + 紫红 |
| 暴风雨 | 乌云密布、闪电、雨幕 | 深灰蓝 + 白闪 |

**视差滚动（Parallax Scrolling）分层：**

每个背景主题必须拆分为至少 3 个独立图层，以不同速度滚动制造纵深感：

| 图层 | 内容示例 | 滚动速度 |
|------|---------|---------|
| 远景（Sky Layer） | 天空渐变、远山轮廓、太阳/月亮 | 最慢（约 0.1x ~ 0.2x） |
| 中景（Mid Layer） | 建筑群、山体、云层 | 中速（约 0.4x ~ 0.6x） |
| 近景（Front Layer） | 树木、路灯、栅栏、电线杆 | 较快（约 0.8x） |
| 前景装饰（可选） | 飘落的树叶、飞鸟 | 最快（1.0x 或更快） |

**天气效果叠加层：**

天气效果作为独立粒子层叠加在背景之上、UI之下：

- **雨滴：** 45度角下落的短线条粒子，半透明蓝白色
- **雪花：** 缓慢飘落的小方块粒子，带轻微左右摆动
- **落叶：** 不规则飘落的彩色小精灵（2-3种叶片形态）
- **风线：** 水平方向的半透明白色线条，表现强风

### 2.3 场景内部（NPC交互场景）

当玩家进入某个地点与NPC互动时，切换到该地点的内部场景。

**设计规范：**

- 画面比例：16:9 横版
- 场景为静态背景图（不可滚动），单屏展示
- 每个地点一张独立的内部背景：
  - **家：** 客厅/书桌场景，暖色灯光，书架上有纸飞机收藏
  - **学校：** 教室场景，黑板上画有折纸图解，课桌整齐排列
  - **后山：** 室外场景，树荫下的空地，远处可见小镇
  - **小卖部：** 柜台场景，货架上陈列各种纸张和零件
  - **公园：** 室外长椅场景，花丛、喷泉为背景
- NPC以**固定站位**方式出现在场景中，位置预设好
- 对话时弹出对话框，NPC的大头像显示在对话框中

## 3. 角色与NPC设计

### 3.1 主角

主角是一名充满好奇心和热情的小学生，玩家的化身。

**外观设计：**

- 年龄感：10-12岁小学生
- 标志性元素：戴一顶鲜明的帽子或头带（作为角色辨识标记）
- 服装：休闲校服或日常便装，简洁不花哨
- 整体配色以暖色为主（橙色帽子、白/蓝衣服等）

**精灵图需求：**

由于节点地图不需要行走动画，主角的精灵需求集中在对话场景：

| 资源 | 尺寸 | 说明 |
|------|------|------|
| 地图图标 | 32x32 px | 在节点地图上表示玩家位置的小图标 |
| 对话大头像 | **64x64** 或 **96x96 px** | 对话框中显示的半身像 |

**表情变化（大头像版本）：**

| 表情 | 使用场景 |
|------|---------|
| 开心/自信 | 获得胜利、得到新飞机/零件 |
| 沮丧/失落 | 比赛失败、错过机会 |
| 惊讶/震惊 | 遇到意外事件、发现秘密 |
| 思考/犹豫 | 面临选择、研究折纸方法 |
| 愤怒/不服 | 被对手挑衅、遇到不公 |
| 默认/平静 | 日常对话 |

### 3.2 五位馆主对手

五位馆主是游戏主线中的关键Boss角色，每位都有鲜明的个性和视觉特征，且与其擅长的飞机类型在视觉风格上形成呼应。

#### 馆主一：速度型

| 属性 | 设计 |
|------|------|
| 关键词 | 速度、冲劲、直率 |
| 发型 | 短发、利落 |
| 服装 | 运动装（田径背心/运动外套） |
| 配色 | **红色系**为主，搭配白色和黑色 |
| 标志元素 | 运动护腕、跑鞋 |
| 视觉呼应 | 与速度型纸飞机（Paper Dart）的锐利线条一致 |
| 性格表达 | 表情果断、姿态前倾 |

#### 馆主二：特技型

| 属性 | 设计 |
|------|------|
| 关键词 | 花哨、创意、不可预测 |
| 发型 | 长发或造型独特的发型 |
| 服装 | 花哨装饰、围巾或飘带 |
| 配色 | **紫色系**为主，搭配粉色和金色 |
| 标志元素 | 手上总拿着一架复杂折纸 |
| 视觉呼应 | 与特技型纸飞机（Butterfly Wing）的多变造型一致 |
| 性格表达 | 表情戏谑、姿态夸张 |

#### 馆主三：稳定型

| 属性 | 设计 |
|------|------|
| 关键词 | 沉稳、可靠、智慧 |
| 发型 | 整齐、中规中矩 |
| 服装 | 衬衫或毛衣背心，整洁端正 |
| 配色 | **蓝色系**为主，搭配灰色和白色 |
| 标志元素 | 眼镜（圆框或方框） |
| 视觉呼应 | 与稳定型纸飞机（Classic Glider）的宽体对称设计一致 |
| 性格表达 | 表情冷静、姿态挺拔 |

#### 馆主四：均衡型

| 属性 | 设计 |
|------|------|
| 关键词 | 全面、适应力强、亲和 |
| 发型 | 普通但有个人特色（如发卡或发带） |
| 服装 | 校服改造（加了DIY装饰） |
| 配色 | **绿色系**为主，搭配黄色和白色 |
| 标志元素 | 书包上挂着各式纸飞机挂件 |
| 视觉呼应 | 外观综合各型飞机特征，无极端倾向 |
| 性格表达 | 表情友善、姿态自然 |

#### 馆主五（最终Boss）

| 属性 | 设计 |
|------|------|
| 关键词 | 传说、王者、神秘 |
| 发型 | 飘逸、有气场 |
| 服装 | 白色长衣/风衣 |
| 配色 | **白色 + 金色点缀**，高级感 |
| 标志元素 | 手持一架金色折痕的白色纸飞机 |
| 视觉呼应 | 纸飞机的极致形态——纯白与金，超越所有类型 |
| 性格表达 | 表情平和但眼神锐利，气场压人 |

**所有馆主的精灵需求：**

- 对话大头像（64x64 或 96x96 px）：至少 3 种表情（默认、战斗前挑衅、战败）
- 地图/比赛UI中的小图标（32x32 px）

### 3.3 普通对手

普通对手使用**模板化设计**，通过组合变化产生丰富的外观多样性。

**模板系统：**

- 预设 **4-6 种基础身体模板**（不同体型/姿态）
- 每种模板搭配**多种配色方案**（4-5种）
- 附加**小部件差异化元素**：帽子、眼镜、发型、围巾等
- 组合产出：6 模板 x 5 配色 x 若干部件 = **20+ 种独立外观**

**普通对手不需要对话大头像**，仅需：

- 32x32 px 小精灵（用于比赛中的对手标识）
- 可选：比赛结果画面中的简单立绘

### 3.4 NPC设计

以下是关键NPC的设计方向：

| NPC | 外观要点 | 性格基调 |
|-----|---------|---------|
| 妈妈 | 围裙、温柔的发型、暖色系服装 | 温暖关怀，鼓励主角 |
| 爸爸 | 休闲装、可能戴眼镜或留胡子 | 稳重支持，偶尔传授折纸技巧 |
| 老师 | 眼镜、手持折纸教材或成品、端庄装束 | 知识渊博，教导折纸基础 |
| 小卖部老板 | 围裙、笑容可掬、店铺帽 | 热情爽快，推销商品 |
| 神秘折纸大师 | 斗篷或斗笠、白胡子、手指灵巧 | 神秘莫测，传授高级技法 |
| 小伙伴A | 活泼装扮，与主角年龄相仿 | 提供情报、并肩作战 |
| 小伙伴B | 安静装扮，可能拿着笔记本 | 分析型，提供数据和建议 |

每位NPC需要：

- 对话大头像（64x64 或 96x96 px）：2-3 种表情
- 场景内站立精灵（32x48 px）：1-2 帧（可含微小idle动画）

## 4. 纸飞机设计

### 4.1 基础设计原则

纸飞机是《纸翼传说》最核心的视觉元素，是玩家投入时间最多、关注度最高的视觉对象。其设计必须遵循以下原则：

1. **辨识度优先：** 每种机型在缩略图尺寸下就必须能明确区分
2. **纸质感贯穿：** 白色为主体色（纸的自然色），通过线条和折痕（像素线条的走向与明暗）表现立体折叠感
3. **类型色彩编码：** 通过强调色快速传达飞机类型（速度=红、特技=紫、稳定=蓝）
4. **动画表现力：** 飞行中的微动画让飞机"活"起来，而非僵硬的静态图片

### 4.2 速度型飞机系列

**造型关键词：** 修长、尖锐、流线型

**典型代表：纸飞镖（Paper Dart）**

| 属性 | 设计说明 |
|------|---------|
| 外形轮廓 | 窄长三角形，机鼻尖锐，翼展较小 |
| 折痕表现 | 中线折痕明显，翼面简洁少折叠 |
| 颜色方案 | 白色机身 + **红色/橙色条纹**强调速度感 |
| 条纹位置 | 沿机翼后缘或机身中线 |
| 视觉传达 | 一看就"快"——锐利、极简、空气阻力感低 |

**飞行动画特征：**

- 飞行姿态笔直，几乎无翻转
- 轻微的机翼颤动（1-2像素幅度）
- 加速时尾部拖出短速度线
- 高速状态下机身略微拉长（拉伸效果1-2帧）

### 4.3 特技型飞机系列

**造型关键词：** 宽翼、蝴蝶形、不规则折痕

**典型代表：蝴蝶翼（Butterfly Wing）**

| 属性 | 设计说明 |
|------|---------|
| 外形轮廓 | 宽大机翼，可能有翼尖上翘或波浪边缘 |
| 折痕表现 | 复杂折叠纹理，翼面有多重折痕 |
| 颜色方案 | 白色机身 + **紫色/粉色装饰**图案 |
| 装饰位置 | 翼面花纹、翼尖渐变色 |
| 视觉传达 | 华丽、灵动、不走寻常路 |

**飞行动画特征：**

- 飞行中有明显的翻转和旋转动作
- 轨迹多变，机翼有较大幅度的上下摆动
- 特技触发时有旋转拖尾特效
- 飞行姿态不断微调，显得"活泼"

### 4.4 稳定型飞机系列

**造型关键词：** 宽体、对称、厚实

**典型代表：经典滑翔机（Classic Glider）**

| 属性 | 设计说明 |
|------|---------|
| 外形轮廓 | 宽阔机身，机翼面积大，整体敦厚 |
| 折痕表现 | 折叠层次感强，看起来"厚重扎实" |
| 颜色方案 | 白色机身 + **蓝色/绿色标识**线 |
| 标识位置 | 翼面边线或机身中段色块 |
| 视觉传达 | 稳重、可靠、不容易失控 |

**飞行动画特征：**

- 平稳滑翔为主，姿态变化极小
- 轻微的上下浮动（呼吸感），幅度1-2像素
- 遇到气流时反应小，不会剧烈摇晃
- 给人"稳如泰山"的视觉感受

### 4.5 精灵图规范

每种纸飞机的精灵图（Sprite Sheet）必须包含以下标准动画状态：

| 动画状态 | 帧数 | 用途说明 |
|---------|------|---------|
| 静止状态（Idle） | 1帧 | 菜单展示、图鉴浏览、构建界面预览 |
| 飞行状态（Fly） | 4-8帧循环 | 比赛中正常飞行，含轻微摆动/尾部抖动 |
| 加速状态（Boost） | 2帧 | 触发加速技能时，附带速度线效果 |
| 失速状态（Stall） | 2帧 | 速度过低时的下坠旋转动画 |
| 碰撞状态（Hit） | 2帧 | 撞击障碍物或被道具击中的反馈 |
| 特殊状态（可选） | 2-4帧 | 特技飞机的翻转、特殊技能演出等 |

**方向约定：**

- 所有精灵图**统一朝右绘制**
- 向左飞行时由代码进行**水平镜像（Flip X）** 处理
- 某些非对称设计（如单侧涂装）需注意镜像后的视觉效果

**导出流程：**

```
Aseprite 制作 → 导出 PNG Spritesheet + JSON Atlas 文件
```

- PNG格式，背景透明
- JSON Atlas 记录每帧的坐标、尺寸、持续时间
- 文件名遵循命名规范（见第7节）

### 4.6 零件视觉差异

零件系统是飞机自定义的核心玩法，**每个零件装配后必须在飞机外观上产生可见变化**。

| 零件槽位 | 视觉影响区域 | 变化示例 |
|---------|-------------|---------|
| 机鼻（Nose） | 飞机前端形状 | 尖锥→圆头→分叉 |
| 机翼（Wing） | 翼型和翼展 | 窄翼→宽翼→锯齿翼→后掠翼 |
| 尾翼（Tail） | 尾部结构 | 无尾翼→V形尾→T形尾→双垂尾 |
| 涂装（Skin） | 整体颜色和花纹 | 纯白→条纹→迷彩→星空→火焰 |
| 配重（Weight） | 机身小标记/贴纸 | 回形针图标→胶带贴纸→硬币标记 |

**实现方式建议：**

- 每个零件对应一个独立的精灵图层（Sprite Layer）
- 在运行时将基础飞机精灵与零件精灵图层叠加合成
- 或者预渲染所有常见组合的合成精灵图（适用于组合数量可控的情况）

## 5. UI设计

### 5.1 全局UI风格

**字体规范：**

- 游戏全局使用**像素字体（Pixel Font）**，中英文统一风格
- 中文像素字体推荐方案：

| 字体 | 特点 | 许可 |
|------|------|------|
| **zpix** | 支持中英文的完整像素字体，风格复古 | 免费开源 |
| **Silver** | 精致像素字体，支持拉丁字符 | 免费 |
| 自制点阵字 | 按需定制，风格最统一 | 需要制作成本 |

- 字号层级：
  - 标题：16px（像素尺寸）
  - 正文/对话：8px 或 10px
  - UI数值：8px
  - 提示/注释：6px

**按钮设计：**

- 形状：圆角矩形（像素圆角，radius约2-3px）
- 边框：1px实线像素边框
- 状态反馈：
  - 默认：正常颜色
  - 悬停（Hover）：边框变亮/按钮微微放大1px
  - 按下（Pressed）：整体下移1-2px，颜色微暗
  - 禁用（Disabled）：灰色半透明

**面板设计：**

- 背景：半透明深色（rgba黑色，60%-80%透明度）
- 边框：1px白色或浅色像素边框
- 圆角：2-3px像素圆角
- 内容与边框保持4-8px内边距

### 5.2 HUD布局（比赛中）

#### 横屏模式（推荐默认）

**布局原则：**

- 顶部信息栏：赛道进度条（水平长条，显示飞机位置）、当前名次、天气图标、计时
- 右侧技能栏：2-3个大按钮，图标清晰，冷却时间用扇形遮罩表示
- 左下Buff列表：当前激活的增益/减益图标（16x16小图标），最多显示4-6个

### 5.3 构建界面（Build Scene）

构建界面是玩家组装和调整纸飞机的核心界面。

**布局设计：**

```
┌──────────┬──────────────────┬──────────┐
│          │                  │          │
│  零件    │   飞机预览大图    │  属性    │
│  列表    │   （居中展示）    │  雷达图  │
│ （可滚动）│                  │（五维图）│
│          │  [鼻] [翼] [尾]  │          │
│          │  [涂装] [配重]   │  速度:██ │
│          │                  │  特技:█  │
│          │                  │  稳定:███│
│          │                  │  ...     │
├──────────┴──────────────────┴──────────┤

```

**交互方式：**

- 左侧零件列表可上下滚动浏览可用零件
- 中间飞机预览实时反映零件更换效果
- 飞机下方/上方显示槽位指示，点击槽位可打开对应零件选择
- 拖拽零件到对应槽位进行装配（触屏优化）
- 右侧属性雷达图（五维：速度、特技、稳定、加速、抗风）实时更新

**属性雷达图：**

- 五边形雷达图，像素线条绘制
- 填充区域使用半透明色
- 更换零件时雷达图有平滑过渡动画
- 数值变化用红/绿色闪烁表示提升/下降

### 5.4 菜单界面

**主菜单：**

- 像素风格的游戏Logo居中靠上（"纸翼传说"四个大字 + Paper Wings Legend英文副标题）
- 背景：一架纸飞机在蓝天中飞翔的简洁动画（循环动画）
- 菜单选项纵向排列：
  - 开始游戏
  - 继续游戏
  - 设置
  - 图鉴
  - 退出
- 选中项有箭头指示或高亮框

**暂停菜单：**

- 半透明黑色遮罩覆盖游戏画面（50%-70%透明度）
- 居中弹出面板：
  - 继续
  - 重新开始
  - 设置
  - 返回主菜单
- 暂停时游戏画面可见但不可操作

## 6. 音效与音乐

### 6.1 音乐风格

游戏音乐以 **8-bit / Chiptune 风格**为核心基调，可以混合少量现代电子音乐元素以提升表现力。

**情绪基调：** 积极向上、充满冒险感、带有童年怀旧的温暖感。

**音乐设计原则：**

- 旋律优先——即使是8-bit音色也要有记忆点
- 循环无缝——所有BGM必须支持无缝循环播放
- 场景适配——音乐的节奏和情绪与场景功能一致
- 渐变过渡——场景切换时音乐淡入淡出，不要硬切

### 6.2 音乐列表

| 场景 | 风格描述 | BPM范围 | 情绪关键词 |
|------|---------|---------|-----------|
| 主菜单 | 轻快、怀旧的旋律 | 100-110 | 温暖、期待、童真 |
| 城镇地图 | 温暖、日常感的背景音乐 | 90-100 | 安心、日常、探索 |
| 锦标赛地图 | 紧张感逐渐递增 | 110-120 | 紧张、期待、挑战 |
| 比赛飞行 | 激昂、快节奏的驱动力音乐 | 130-150 | 激动、速度、竞争 |
| Boss战 | 史诗感、紧迫感的强节奏 | 140-160 | 史诗、紧迫、决战 |
| 胜利结算 | 欢快短曲（约10-15秒） | ~120 | 喜悦、成就、骄傲 |
| 失败结算 | 温和短曲（不让玩家沮丧） | ~80 | 鼓励、不气馁、再接再厉 |
| 折纸教学 | 轻柔、专注感的环境音乐 | 80-90 | 专注、学习、好奇 |
| 商店 | 俏皮、轻松的购物感 | 100-110 | 愉快、休闲、期待 |

## 7. 资源管理

### 7.1 精灵图集打包规范

**打包工具：**

- 推荐使用 **TexturePacker** 或免费替代 **free-tex-packer**
- 输出格式：PNG图集 + JSON数据文件（Phaser / PixiJS兼容格式）

**图集分组策略——按功能拆分：**

| 图集文件 | 内容 | 最大尺寸 |
|---------|------|---------|
| `characters.png` + `characters.json` | 所有角色精灵（主角、NPC、对手） | 2048x2048 px |
| `airplanes.png` + `airplanes.json` | 所有纸飞机精灵及零件 | 2048x2048 px |
| `ui.png` + `ui.json` | UI元素（按钮、面板、图标） | 2048x2048 px |
| `effects.png` + `effects.json` | 特效精灵（烟尘、风线、粒子、速度线） | 1024x1024 px |
| `backgrounds/` | 背景图层（单独文件，不打包） | 按需 |

**打包注意事项：**

- 单张图集尺寸**不超过 2048x2048 像素**（兼容低端设备）
- 精灵之间留 **1px 间距（padding）**，避免纹理采样时出现渲染伪影（bleeding artifact）
- 开启**去除多余透明像素（trim）** 优化图集空间利用率
- 导出时使用 **premultiply alpha** 避免半透明边缘白边

### 7.2 动画帧命名规范

所有动画帧文件名遵循统一命名格式，确保代码中可以通过字符串拼接程序化加载：

**格式：** `{entity}_{variant}_{state}_{frame}`

**示例：**

| 分类 | 命名示例 | 说明 |
|------|---------|------|
| 纸飞机 | `airplane_dart_fly_01` | 纸飞镖-飞行状态-第1帧 |
| 纸飞机 | `airplane_dart_fly_02` | 纸飞镖-飞行状态-第2帧 |
| 纸飞机 | `airplane_glider_idle_01` | 经典滑翔机-静止状态-第1帧 |
| 纸飞机 | `airplane_butterfly_stall_01` | 蝴蝶翼-失速状态-第1帧 |
| 主角 | `player_default_happy_01` | 主角-默认造型-开心表情 |
| NPC | `npc_mom_happy_01` | 妈妈-开心表情-第1帧 |
| NPC | `npc_rival1_angry_01` | 馆主1-愤怒表情-第1帧 |
| NPC | `npc_master_idle_01` | 折纸大师-待机状态-第1帧 |
| 特效 | `fx_speedline_01` | 速度线特效-第1帧 |
| 特效 | `fx_smoke_01` | 烟尘特效-第1帧 |
| UI | `ui_btn_confirm_normal` | 确认按钮-正常状态 |
| UI | `ui_btn_confirm_hover` | 确认按钮-悬停状态 |
| UI | `ui_btn_confirm_pressed` | 确认按钮-按下状态 |

**命名规则总结：**

- 全部使用**小写字母**
- 单词之间用**下划线 `_`** 分隔
- 帧编号使用**两位数字**（01, 02, ... 99）
- 状态名使用统一的英文关键词：`idle`, `fly`, `boost`, `stall`, `hit`, `happy`, `angry`, `sad`, `normal`, `hover`, `pressed`, `disabled`

### 7.5 资源制作优先级

为了配合开发进度，美术资源的制作应按以下优先级分批推进：

**第一优先级（核心原型阶段）：**

- 1种纸飞机的完整精灵图（含所有动画状态）
- 1个比赛背景（蓝天白云默认主题，含视差图层）
- 基础UI元素（按钮、面板、HUD框架）
- 主角对话大头像（2-3种表情）
- 基础音效（发射、碰撞、UI点击）

**第二优先级（玩法完善阶段）：**

- 其余纸飞机类型的精灵图
- 节点地图底图与地点图标
- 五位馆主的对话大头像
- 更多比赛背景主题
- 背景音乐（主菜单、比赛飞行）
- 构建界面的完整UI

**第三优先级（内容丰富阶段）：**

- NPC全套精灵与表情
- 普通对手的模板化精灵
- 天气特效粒子
- 零件视觉差异精灵
- 全部音乐与音效
- 场景内部背景

**第四优先级（打磨阶段）：**

- 昼夜变化效果
- 过渡动画与演出效果
- 成就/图鉴相关美术素材
- 多语言字体适配
- 视觉特效优化
